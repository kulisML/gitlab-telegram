# Конфигурация

Все настройки бота задаются через файл `.env` в корне проекта.

## Переменные окружения

### Обязательные

| Переменная | Описание | Пример |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота от [@BotFather](https://t.me/BotFather) | `1234567890:AAExampleToken` |
| `TELEGRAM_CHAT_ID` | ID чата или группы для уведомлений | `1196267761` или `-1001234567890` |

### Необязательные

| Переменная | По умолчанию | Описание |
|---|---|---|
| `GITLAB_SECRET_TOKEN` | *(пусто)* | Секретный токен для верификации запросов от GitLab |
| `APP_HOST` | `0.0.0.0` | IP-адрес для прослушивания |
| `APP_PORT` | `8000` | Порт HTTP-сервера |
| `DATA_DIR` | `data/webhooks` | Папка для JSON-архива событий |

---

## Порты

| Порт | Когда использовать |
|---|---|
| **80** | Стандартный HTTP. GitLab может слать запросы напрямую. Требует прав администратора. |
| **8000** | Без прав администратора. Нужен NGrok или reverse proxy. |
| **Любой другой** | При работе за nginx/caddy — задайте любой внутренний порт. |

---

## Пример полного `.env`

```env
# ─── TELEGRAM ─────────────────────────────────────────────
# Токен бота — получить у @BotFather → /newbot
TELEGRAM_BOT_TOKEN=1234567890:AAExampleTokenHere

# ID чата/группы куда слать уведомления
TELEGRAM_CHAT_ID=1196267761

# ─── GITLAB ───────────────────────────────────────────────
# Секретный токен — одинаковый в .env и в настройках Webhook GitLab
GITLAB_SECRET_TOKEN=MySecret123

# ─── SERVER ───────────────────────────────────────────────
# Папка для JSON-архива всех входящих событий
DATA_DIR=data/webhooks

# IP для прослушивания (0.0.0.0 = все сетевые интерфейсы)
APP_HOST=0.0.0.0

# Порт приложения (80 = стандартный HTTP)
APP_PORT=80
```

## Настройки через Telegram-команды

После запуска бота часть настроек доступна прямо из Telegram командой `/tgbot`:

| Настройка | Как изменить |
|---|---|
| Время ежедневного отчёта | `/tgbot` → **📅 Расписание** → введите `ЧЧ:ММ` |
| Включённые типы событий | `/tgbot` → **🔔 Типы событий** → кнопки-переключатели |
| Привязка GitLab username | `/tgbot` → **🔗 Мой GitLab** → введите username |

Эти настройки хранятся в SQLite БД (`bot.db`) и сохраняются между перезапусками.

---

## База данных

Бот автоматически создаёт файл `bot.db` при первом запуске. Файл содержит три таблицы:

| Таблица | Что хранит |
|---|---|
| `users` | Telegram ID, username, привязанный GitLab username |
| `events` | Все входящие webhook-события с метаданными |
| `settings` | Время отчёта, список включённых событий |

Резервная копия — просто скопируйте `bot.db`.

# Устранение неисправностей

## Бот не запускается

### `TokenValidationError: Token is invalid`
**Причина:** `TELEGRAM_BOT_TOKEN` не задан или неверен.

**Решение:**
1. Проверьте, что файл `.env` существует в корне проекта
2. Убедитесь, что токен скопирован полностью (без пробелов)
3. Проверьте, что бот ещё активен у [@BotFather](https://t.me/BotFather): `/mybots`

---

### `Permission denied` на порту 80
**Причина:** Порт 80 требует прав администратора.

**Решение (Linux):**
```bash
sudo python3 main.py
# или
sudo setcap 'cap_net_bind_service=+ep' /usr/bin/python3
```

**Решение:** Измените порт на `8000` в `.env` и используйте NGrok или Nginx.

---

### `Address already in use`
**Причина:** Порт уже занят другим процессом.

```bash
# Найти процесс на порту 80
sudo lsof -i :80

# Или на 8000
sudo lsof -i :8000

# Завершить процесс
sudo kill -9 <PID>
```

## GitLab не доставляет события

### Webhook возвращает `403 Forbidden`
**Причина:** `X-Gitlab-Token` не совпадает.

**Решение:**
1. Проверьте `GITLAB_SECRET_TOKEN` в `.env`
2. Убедитесь, что в настройках GitLab Webhook указан **точно такой же** токен

---

### Webhook возвращает `Connection refused`
**Причина:** Бот недоступен из интернета.

**Решение:**
- Если сервер за NAT/firewall — настройте [NGrok](./deployment.md#вариант-2-ngrok-быстро-для-тестов)
- Если есть белый IP — проверьте, что порт открыт в firewall:
  ```bash
  sudo ufw allow 80/tcp
  ```

---

### Сообщения не приходят в Telegram

**Шаг 1:** Проверьте health-check:
```bash
curl http://localhost:80/health
```

**Шаг 2:** Убедитесь, что бот добавлен в чат и имеет права на отправку сообщений.

**Шаг 3:** Проверьте `TELEGRAM_CHAT_ID` — для группы он начинается с `-100`.

**Шаг 4:** Посмотрите логи на ошибки Telegram API:
```bash
# При запуске через systemd
sudo journalctl -u gitlab-bot -n 50
```

---

### Событие пришло, но уведомления нет

**Причина 1:** Тип события выключен в настройках.
Проверьте: `/tgbot` → **🔔 Типы событий**.

**Причина 2:** Для события нет форматтера (неизвестный тип).
Посмотрите в логи — будет запись `ignored (no formatter)`.

**Причина 3:** Форматтер вернул пустую строку.
Например, Pipeline с `status=running` не генерирует уведомление (намеренно).

---

## @Упоминания не работают

**Причина:** Пользователь не зарегистрировался в боте или не привязал GitLab username.

**Решение:**
1. Напишите `/tgbot` в чате
2. Нажмите **🔗 Мой GitLab**
3. Введите username из GitLab (без `@`)

---

## Ежедневный отчёт не приходит

1. Проверьте, что время настроено правильно: `/tgbot` → **📅 Расписание**
2. Бот использует **Московское время** (`Europe/Moscow`)
3. Убедитесь, что за прошедший день были события (пустой день тоже присылается)
4. Проверьте, что бот запущен в момент отправки отчёта

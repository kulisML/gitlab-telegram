# Поддерживаемые события GitLab

Бот поддерживает 8 типов событий. Каждое событие форматируется
в информативное Telegram-сообщение с @упоминанием ответственного.

---

## 🚀 Push — коммиты в ветку

**Когда:** кто-то делает `git push` в любую ветку репозитория.

**Что показывает:**
- Название проекта (со ссылкой)
- @Автор (`@username`, если зарегистрирован в боте)
- Ветка
- Количество коммитов
- До 5 коммитов со ссылками, сообщением и числом изменённых файлов

**Пример сообщения:**
```
🚀 Пуш в репозиторий: MyProject
👤 Автор: @ivan_tg
🌿 Ветка: main
📦 Коммитов: 3

Коммиты:
- [abc12345] feat: добавлена авторизация (+5, ~2 файл(ов))
- [def67890] fix: исправлен баг при логине (~1 файл(ов))
- [aaa11111] refactor: очистка кода (-3 файл(ов))
```

---

## 🏷️ Tag Push — новый тег / релиз

**Когда:** создаётся новый тег (`git tag v1.0.0 && git push --tags`).

**Что показывает:** тег со ссылкой, проект, автор, сообщение к тегу.

---

## 🔀 Merge Request

**Когда:** MR открыт, закрыт или смёрджен.

| Действие | Эмодзи |
|---|---|
| Открыт (`open`) | 🆕 |
| Смёрджен (`merge`) | ✅ |
| Закрыт (`close`) | ❌ |
| Переоткрыт (`reopen`) | 🔄 |

**Что показывает:** заголовок MR (со ссылкой), проект, инициатор, ветки `source → target`.

---

## 🚦 Pipeline — CI/CD пайплайн

**Когда:** пайплайн завершается успехом (`success`) или падением (`failed`).

> Промежуточные статусы (`pending`, `running`) **игнорируются** — не засоряют чат.

**Что показывает:** ID пайплайна (со ссылкой), проект, ветка, время выполнения, автор коммита.

---

## 📕 Issue — задача

**Когда:** задача открыта, закрыта или переоткрыта.

**Что показывает:** название задачи (со ссылкой), проект, автор, краткое описание (до 150 символов).

---

## 💬 Note — комментарий

**Когда:** кто-то оставляет комментарий к MR, Issue или коммиту.

**Что показывает:** тип объекта (к чему комментарий), проект, автор, текст (до 200 символов).

---

## 🔧 Build / Job — задача CI/CD

**Когда:** отдельная задача в пайплайне завершается успехом или падением.

**Что показывает:** название задачи (`test:unit`, `deploy:prod`), проект, исполнитель, время.

---

## 📖 Wiki Page — страница Wiki

**Когда:** страница Wiki создана, обновлена или удалена.

**Что показывает:** название страницы (со ссылкой), проект, автор.

---

## Управление событиями

Включить или выключить любой тип событий можно без перезапуска бота:

`/tgbot` → **🔔 Типы событий**

Выключенные события **не теряются** — они сохраняются в базе данных и учитываются в статистике, просто не отправляются в Telegram.
